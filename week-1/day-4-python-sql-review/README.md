# Day 4 - Python and SQL Logic Review

## Goal of the practice

The goal of this practice was to understand how the same data processing tasks can be done using both Python and SQL.

I worked with an order dataset and practiced:

* Filtering data
* Sorting records
* Creating calculated values
* Finding business insights
* Comparing Python logic with SQL logic

## Files included

* `python_orders.py` - Python solution using a list of dictionaries
* `setup.sql` - SQL table creation and data insertion
* `sql_queries.sql` - SQL queries for filtering, sorting, and calculations
* `comparison.md` - Comparison between Python and SQL approaches
* `daily_report.txt` - Summary of my practice day
* `screenshots/` - SQL and terminal screenshots

## How to run Python

Run the Python file with:

```bash
python python_orders.py
```

The program will display:

* Total number of orders
* Filtered orders by status
* Calculated total amounts
* Sorted orders
* Revenue calculations
* Business challenge results

## SQL Practice

SQL was practiced using SQLiteOnline.com.

The SQL tasks included:

* Creating the orders table
* Inserting order data
* Using SELECT and WHERE
* Sorting with ORDER BY
* Limiting results with LIMIT
* Creating calculated columns
* Using SUM, COUNT, and GROUP BY

## What I learned

From this practice I learned that Python and SQL can solve similar data problems.

Python uses:

* Lists
* Dictionaries
* Loops
* Conditions
* Functions

SQL uses:

* Tables
* SELECT
* WHERE
* ORDER BY
* Calculated columns

Both tools can be used to filter data, calculate values, and create useful business reports.
