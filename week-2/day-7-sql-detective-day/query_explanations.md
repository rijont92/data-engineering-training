# Query Explanations - Day 7 SQL Detective Day

## Fixed Query 1 - Count orders by city

### What the query does:
This query counts how many orders exist in each city.

### Tables used:
- orders
- customers

### Why JOIN is needed:
The city column is not stored in the orders table. The customers table contains city information, so a JOIN is needed using customer_id.

### What the result means:
The result shows which cities have the highest number of orders and where customer activity is concentrated.

## Fixed Query 2 - Completed revenue by product name

### What the query does:
This query calculates the total completed revenue generated by each product.

### Tables used:
- orders
- products

### Why JOIN is needed:
The product name and price are stored in the products table, while quantity and order status are stored in orders. JOIN combines this information.

### What the result means:
The result shows which products generate the most revenue from completed orders.
